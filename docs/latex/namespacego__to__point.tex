\begin{DoxyVerb}Description of the callback:
The action client defined in state_machine.cpp fill the values stored inside goal.
These represent the random position that the robot reaches. 
The desired_position is initialised as the goal of the action. 
The feedback of the action message is constantly updated as the current pose of 
the robot. To conclude with, the change_state function (which controls
the _state of the robot) is given with argument zero so that the robot 
can start fixing its own yaw in the right orientation in front of the robot\end{DoxyVerb}
